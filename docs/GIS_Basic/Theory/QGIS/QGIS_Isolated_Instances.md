# QGIS-ის იზოლირებული პროცესები Windows-ში

## რატომ არის ეს კრიტიკულად მნიშვნელოვანი?

QGIS-ის შემთხვევაში დამოუკიდებელი პროცესები განსაკუთრებულ მნიშვნელობას იძენს, რადგან დიდ ვექტორულ ფენებთან, კომპლექსურ გეოპროცესინგთან ან მძიმე რასტრებთან მუშაობისას ერთი პროექტის გაჭედვამ (მაგალითად, Crash-მა რენდერინგის დროს) შეიძლება მთელი დღის ნამუშევარი ჩაიყოლოს სხვა ფანჯრებშიც.

ნაგულისხმევად (Default), Windows-ში QGIS-ის ახალ ფანჯარას გახსნისას ოპერაციული სისტემა მას ხშირად უკვე არსებულ მთავარ პროცესს აბამს.

იმისათვის, რომ ყოველი გახსნილი QGIS იყოს 100%-ით დამოუკიდებელი, არსებობს ორი საუკეთესო გზა.

---

## მეთოდი 1: მალსახმობის (Shortcut) მოდიფიკაცია — ყველაზე საიმედო

QGIS-ს აქვს ჩაშენებული ბრძანება `--no-version-check`, რომელიც აიძულებს პროგრამას, Windows-ის რეესტრის გვერდის ავლით, სრულიად სუფთა, იზოლირებული ახალი სესია გაუშვას.

### ნაბიჯები:

**ნაბიჯი 1 — მალსახმობის თვისებების გახსნა**

Desktop-ზე ან Start მენიუში იპოვნეთ QGIS-ის აიქონი, დააწკაპუნეთ მაუსის მარჯვენა ღილაკით და აირჩიეთ **Properties (თვისებები)**.

**ნაბიჯი 2 — Target ველის პოვნა**

გამოსულ ფანჯარაში დარწმუნდით, რომ ხართ **Shortcut** ჩანართზე და იპოვნეთ ველი სახელად **Target**. იქ დაგხვდებათ დაახლოებით ასეთი მისამართი:

```
"C:\Program Files\QGIS 3.xx.x\bin\qgis-bin.exe"
```

**ნაბიჯი 3 — არგუმენტის დამატება**

მიიტანეთ კურსორი ტექსტის სულ ბოლოში, დასვით ერთი გამოტოვება (Space) ბრჭყალების შემდეგ და ჩაწერეთ:

```
--no-version-check
```

საბოლოო სახე ასეთი უნდა იყოს:

```
"C:\Program Files\QGIS 3.xx.x\bin\qgis-bin.exe" --no-version-check
```

**ნაბიჯი 4 — ცვლილების შენახვა**

დააჭირეთ **Apply** და შემდეგ **OK**. თუ სისტემამ ითხოვა ადმინისტრატორის ნებართვა, დაადასტურეთ (Continue).

ამიერიდან, ყოველ ჯერზე, როცა ამ მალსახმობით გახსნით QGIS-ს (1-ელ, მე-2 თუ მე-5 ფანჯარას), თითოეული მათგანი იქნება სრულიად დამოუკიდებელი პროცესი.

---

## მეთოდი 2: გახსნა პირდაპირ მენიუდან — Project → New Window

თუ უკვე მუშაობთ QGIS-ში და გსურთ მეორე პროექტის პარალელურად გახსნა:

1. ზედა მენიუში დააჭირეთ **Project**-ს.
2. აირჩიეთ **New Window** (და არა უბრალოდ **New**, რომელიც მიმდინარე ფანჯარას ასუფთავებს).

Windows-ის გარემოში QGIS ამ ბრძანებით ავტომატურად ქმნის ახალ იზოლირებულ **PID (Process ID)** ნომერს ოპერატიულ მეხსიერებაში.

---

## მეთოდი 3: .bat ფაილი — ყოველთვის იზოლირებულად გაშვება

ყველაზე მოქნილი გზაა სპეციალური `.bat` ფაილის შექმნა, რომელიც QGIS-ს ყოველთვის ახალ, იზოლირებულ პროცესად გაუშვებს — პარამეტრების ხელით შეცვლის გარეშე.

**შექმენით ფაილი `qgis_isolated.bat` ასეთი შიგთავსით:**

```bat
@echo off
start "" "C:\Program Files\QGIS 3.xx.x\bin\qgis-bin.exe" --no-version-check
```

> `start ""` ბრძანება Windows-ს ეუბნება, გაუშვას პროგრამა სრულიად ახალ, დამოუკიდებელ პროცესში. ორი ცარიელი ბრჭყალი (`""`) არის ფანჯრის სათაური — მისი გამოტოვება შეუძლებელია, წინააღმდეგ შემთხვევაში Windows მისამართს სათაურად ჩათვლის.

**გამოყენება:**

- შეინახეთ ფაილი Desktop-ზე ან სასურველ ადგილას.
- ორჯერ დააწკაპუნეთ — QGIS გაიხსნება ახალ, 100%-ით იზოლირებულ პროცესად.
- ყოველი გაშვება ახალ, დამოუკიდებელ PID-ს ქმნის.

## გაჭედილი QGIS-ის დახურვა დანარჩენების დაზიანების გარეშე

თუ მუშაობისას ერთ-ერთმა ფანჯარამ "გიპასუხათ" (**Not Responding**), გამოიყენეთ Task Manager-ის **Details** ჩანართი:

1. გახსენით Task Manager: **Ctrl + Shift + Esc**
2. გადადით **Details** ჩანართზე
3. მოძებნეთ **`qgis-bin.exe`** — დაინახავთ რამდენიმე ერთნაირ ჩანაწერს
4. **CPU** ან **Memory** სვეტით განსაზღვრეთ გაჭედილი პროცესი (ჩვეულებრივ 0%-ზე გაყინულია ან პროცესორს ტვირთავს მაქსიმუმზე)
5. დააწკაპუნეთ **მხოლოდ იმ ერთ** პროცესზე მარჯვენა ღილაკით → **End task**

> ⚠️ **მნიშვნელოვანია:** არ გამოიყენოთ **Processes** ჩანართი — იქ Windows-ი აჯგუფებს პროგრამებს და "End Task"-ზე დაჭერით შესაძლოა **ყველა** გახსნილი QGIS ერთად დაიხუროს. **Details** ჩანართიდან კი მხოლოდ ერთი, თქვენ მიერ მონიშნული პროცესი იხურება.
